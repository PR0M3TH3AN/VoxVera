# विस्तृत उपयोग

यह गाइड सामान्य CLI वर्कफ्लो को कवर करती है। डॉकर निर्देशों के लिए `docs/docker.md` और उपलब्ध फ्लायर टेम्पलेट्स के लिए `docs/templates.md` देखें।

## पूर्वापेक्षाएँ (Prerequisites)

VoxVera को अत्यधिक पोर्टेबल बनाने के लिए डिज़ाइन किया गया है और इसके लिए न्यूनतम सिस्टम निर्भरता की आवश्यकता होती है।

### 1. स्टैंडअलोन बाइनरी (अनुशंसित)
आप अपने ऑपरेटिंग सिस्टम के लिए स्टैंडअलोन, शून्य-निर्भरता बाइनरी डाउनलोड कर सकते हैं:
- **Linux:** `voxvera-linux`
- **Windows:** `voxvera-windows.exe`
- **macOS:** `voxvera-macos`

इन बाइनरी में VoxVera चलाने के लिए आवश्यक सब कुछ शामिल है (`onionshare-cli` को छोड़कर)।

### 2. वन-लाइन इंस्टॉलर
वैकल्पिक रूप से, हमारी स्वचालित स्क्रिप्ट के माध्यम से इंस्टॉल करें:

```bash
curl -fsSL https://raw.githubusercontent.com/PR0M3TH3AN/VoxVera/main/install.sh | bash
```

### 3. मैनुअल पायथन इंस्टॉलेशन
यदि आप सोर्स से चलाना पसंद करते हैं:

```bash
pipx install 'voxvera@git+https://github.com/PR0M3TH3AN/VoxVera.git@main'
sudo apt install tor onionshare-cli   # Debian/Ubuntu
```

## चरण-दर-चरण (Step-by-Step)

1. **आरंभ करें (Initialize):** `voxvera init` चलाएँ और संकेतों का पालन करें। आपसे सबसे पहले अपनी भाषा चुनने के लिए कहा जाएगा।
2. **बिल्ड (Build):** फ्लायर एसेट्स जेनरेट करें। प्रत्येक बिल्ड स्वचालित रूप से फ्लायर के फ़ोल्डर में एक `voxvera-portable.zip` बनाता है, जिससे अन्य लोग सीधे आपके फ्लायर से पूरा टूल डाउनलोड कर सकते हैं।
   ```bash
   voxvera build
   ```
3. **सर्व (Serve):** फ्लायर को Tor पर प्रकाशित करें:
   ```bash
   voxvera serve
   ```
   यह स्वचालित रूप से आपके Tor इंस्टेंस का पता लगाता है, OnionShare शुरू करता है, और जेनरेट किए गए .onion एड्रेस को फ्लायर के टियर-ऑफ लिंक में लिख देता है।

## भाषा समर्थन (Language Support)

VoxVera पूरी तरह से स्थानीयकृत (localized) है। आप इंटरैक्टिव सेलेक्टर या सीधे शॉर्टकट का उपयोग करके अपनी भाषा प्राथमिकता को स्थायी रूप से बदल सकते हैं:

- **इंटरैक्टिव सेलेक्टर:** `voxvera lang`
- **सीधा शॉर्टकट:** `voxvera --lang hi` (प्राथमिकता को हिन्दी पर सेट करता है)

### समर्थित भाषाएं:
- **अंग्रेजी:** `en`
- **स्पेनिश:** `es` (उपनाम: `--idioma`)
- **जर्मन:** `de` (उपनाम: `--sprache`)
- **हिन्दी:** `hi`

आप अपनी स्थायी प्राथमिकता को बदले बिना किसी एक कमांड के लिए एक विशिष्ट भाषा को बाध्य (force) भी कर सकते हैं:
- **अंग्रेजी:** `voxvera --lang en check`
- **हिन्दी:** `voxvera --lang hi check`

जेनरेट किए गए फ्लायर्स स्वचालित रूप से विज़िटर की ब्राउज़र भाषा का पता लगाते हैं और उसके अनुसार UI टेक्स्ट बदलते हैं।

## सर्वर प्रबंधन (Server Management)

एक ही इंटरैक्टिव मेनू से कई फ्लायर्स और उनकी Tor पहचान प्रबंधित करें:

```bash
voxvera manage
```

विशेषताएं:
- **{{t('cli.manage_create_new')}}**: पूरा सेटअप अनुक्रम शुरू करें।
- **{{t('cli.manage_start_all')}}**: अपने बेड़े के सभी फ्लायर्स को एक साथ शुरू या बंद करें।
- **वास्तविक समय स्थिति**: सक्रिय `.onion` URL और Tor बूटस्ट्रैपिंग प्रगति संकेतक देखें।
- **व्यक्तिगत नियंत्रण**: विशिष्ट साइटों को ZIP में {{t('cli.manage_action_export')}} करें या उन्हें हटा दें।

## यूनिवर्सल मिररिंग (वायरल वितरण)

यह सुनिश्चित करने के लिए कि VoxVera केंद्रीय रिपॉजिटरी सेंसर होने पर भी सुलभ रहे, प्रत्येक फ्लायर टूल के लिए मिरर के रूप में कार्य करता है।

जब आप फ्लायर होस्ट करते हैं, तो लैंडिंग पेज पर **"{{t('web.download_button')}}"** बटन एक `voxvera-portable.zip` प्रदान करता है जिसमें शामिल हैं:
- पूरा सोर्स कोड और सभी समर्थित भाषाएं।
- सभी पायथन निर्भरताएँ (pre-vendored)।
- क्रॉस-प्लेटफ़ॉर्म Tor बाइनरी।

यह आपके फ्लायर को स्कैन करने वाले किसी भी व्यक्ति को VoxVera टूल का नया वितरक बनने की अनुमति देता है।

## निर्यात और बैकअप (Export & Backup)

अपनी अनूठी Tor पहचान का बैकअप लें (ताकि आपका `.onion` URL कभी न बदले) या अपने फ्लायर्स को दूसरी मशीन पर ले जाएं।

- **एकल साइट निर्यात करें**: `voxvera export <folder_name>`
- **सभी साइटें निर्यात करें**: `voxvera export-all`

**भंडारण स्थान:** सभी निर्यात सभी प्लेटफ़ॉर्म पर उपयोगकर्ता की होम डायरेक्टरी (`~/voxvera-exports/`) में ZIP फ़ाइलों के रूप में सहेजे जाते हैं।

## आयात और पुनर्प्राप्ति (Import & Recovery)

अपनी ZIP फ़ाइलों को `~/voxvera-exports/` में ले जाकर और चलाकर एक नई मशीन पर अपना पूरा सेटअप पुनर्स्थापित करें:

```bash
voxvera import-multiple
```

## पोर्टेबिलिटी और ऑफलाइन उपयोग

यदि आपको बिना इंटरनेट एक्सेस वाली मशीन पर VoxVera चलाने की आवश्यकता है, तो आप पहले निर्भरताओं को "वेंडराइज़" (vendorize) कर सकते हैं:

```bash
voxvera vendorize
```

यह `voxvera/vendor/` में सभी आवश्यक पायथन लाइब्रेरी डाउनलोड करता है। टूल फिर इन स्थानीय फ़ाइलों को प्राथमिकता देगा, जिससे यह `pip install` के बिना चल सकेगा।

## बैच आयात (JSON)

कई JSON कॉन्फ़िगरेशन फ़ाइलों से थोक में फ्लायर्स जेनरेट करने के लिए, उन्हें `imports/` डायरेक्टरी में रखें और चलाएँ:

```bash
voxvera batch-import
```

## URL कैसे काम करते हैं

प्रत्येक फ्लायर के दो अलग-अलग URL होते हैं:
- **टियर-ऑफ लिंक** (स्वचालित रूप से जेनरेट): वह .onion एड्रेस जहाँ फ्लायर होस्ट किया गया है।
- **सामग्री लिंक** (उपयोगकर्ता-कॉन्फ़िगर): वेबसाइट, वीडियो या डाउनलोड की ओर इशारा करने वाला एक बाहरी URL।

आपको मैन्युअल रूप से .onion एड्रेस दर्ज करने की आवश्यकता नहीं है; `serve` चरण के दौरान VoxVera इसे स्वचालित रूप से संभालता है।
